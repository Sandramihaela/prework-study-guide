HTML

  • HTML is the structure/base of our website.
  • By using elements on HTML, we can give meaning to elements as well as control the font, use mark to highlight text and using different size headings and many more.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.

CSS

  • CSS is the interactivity/paint and style of our website.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Some good principles of CSS building and maintaining are consistency, Succinctness, and separation.
    • Stay consistent with your rules! Stick to your decided variations, framework and methodology and be predictable and consistent.
    • Being succinct means being brief and clear, and both are imperative to writing good CSS code.
    • Try to keep three questions in mind when implementing CSS:
      How many classes are needed to achieve the style?
      How long and complex are each of the classes?
      Are the class names easily understandable without context?

    • A Basic CSS rule pattern: "selector{ property: value; }"

Git

  • git status: checks what branch we are currently on.
  • git Checkout -b branch-name: creates a new branch and switches to it.
  • git Pull command is used to get updates from the remote repo.
  • git Add include the changes of a file(s) into our next commit.
  • git Commit Git commit saves your changes only locally and you can get back to it if needed.
  • Git push only uploads changes that are already committed, use: git push -u origin.
  • Git flow commands are in the following order:
    1. git add -A
    2. git commit -m “updated project file”
    3. git pull origin main
    4. git push origin "branch name"
    5. pull request on GitHub
    6. Merge feature branch into main branch on GitHub

JavaScript

  • "let"allows you to declare variables that are block scoped, unlike the var keyword, which is function scoped.
  • Variable declared by let cannot be redeclared and must be declared before use whereas variables declared with var keyword are hoisted.
    • A variable is a named container that allows us to store data in our code.
    • Control flow is the order in which a computer executes code in a script.
